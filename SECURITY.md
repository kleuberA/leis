Security Policy
Supported Versions

The following versions of leis are currently supported with security updates:

Version	Supported
5.1.x	✅
5.0.x	❌
4.0.x	✅
< 4.0	❌

Only versions marked as supported will receive security patches.
Users are strongly encouraged to upgrade to a supported version.

Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

🔒 How to Report

Please do not open a public issue for security vulnerabilities.

Instead, report privately through one of the following methods:

Open a GitHub Security Advisory via the repository's "Security" tab.

Or email directly: [SEU_EMAIL_AQUI]

If possible, include:

A clear description of the vulnerability

Steps to reproduce

Impact assessment

Suggested mitigation (if available)

A proof-of-concept (if applicable)

The more detailed the report, the faster we can triage and respond.

Response Timeline

Initial acknowledgment: within 72 hours

Status update: within 7 days

Fix timeline: depends on severity and complexity

Critical vulnerabilities will be prioritized.

Disclosure Policy

Vulnerabilities will be validated before public disclosure.

Once confirmed and patched, a security advisory may be published.

We follow a responsible disclosure model.

We kindly ask reporters to avoid public disclosure until a fix is available.

Security Considerations

Because this project involves parsing and processing legal documents:

Always validate and sanitize external inputs.

Avoid executing untrusted content.

Ensure scraping processes respect target platform terms of service.

Keep dependencies updated to mitigate supply-chain vulnerabilities.

Scope

This policy applies only to the official repository:

https://github.com/kleuberA/leis

Forks and third-party integrations are not covered under this policy.
